## 🕐 时间处理说明

### 核心原理

本系统的时间处理**完全不依赖网络**，工作原理如下：

```
1. 读取服务器系统时间
2. 获取系统时区偏移量
3. 计算UTC时间 = 系统时间 + 时区偏移
4. 计算北京时间 = UTC时间 + 8小时
```

### 关键特性

✅ **不需要网络连接** - 所有计算都在本地完成  
✅ **不依赖NTP时间同步** - 不需要联网校时  
✅ **不依赖系统时区设置** - 无论系统设置什么时区都能正确计算  
⚠️ **需要系统时间准确** - 系统时间必须设置正确

### 示例说明

假设当前真实北京时间是：**2026年1月27日 21:00**

#### 场景1：系统时区是北京时间（UTC+8）
```
系统显示时间: 2026-01-27 21:00
系统时区偏移: -480分钟（UTC+8）
计算过程:
  UTC时间 = 21:00 + 8小时偏移 = 13:00 UTC
  北京时间 = 13:00 UTC + 8小时 = 21:00 ✅
```

#### 场景2：系统时区是UTC+0（格林威治时间）
```
系统显示时间: 2026-01-27 13:00
系统时区偏移: 0分钟（UTC+0）
计算过程:
  UTC时间 = 13:00 + 0小时偏移 = 13:00 UTC
  北京时间 = 13:00 UTC + 8小时 = 21:00 ✅
```

#### 场景3：系统时区是UTC-8（美国西海岸）
```
系统显示时间: 2026-01-27 05:00
系统时区偏移: +480分钟（UTC-8）
计算过程:
  UTC时间 = 05:00 - 8小时偏移 = 13:00 UTC
  北京时间 = 13:00 UTC + 8小时 = 21:00 ✅
```

**结论**：无论系统设置什么时区，只要系统时间准确，计算结果都正确！

## 📋 部署步骤

### 1. 检查系统时间

在部署前，确保服务器系统时间准确：

#### Windows系统
```powershell
# 查看当前时间
Get-Date

# 手动设置时间（如果需要）
Set-Date -Date "2026-01-27 21:00:00"
```

#### Linux系统
```bash
# 查看当前时间
date

# 手动设置时间（如果需要）
sudo date -s "2026-01-27 21:00:00"
```

### 2. 验证时间处理

运行测试脚本验证时间计算是否正确：

```bash
# 进入后端目录
cd Backed

# 运行内网环境测试
node test-offline-timezone.js
```

**预期输出**：
```
=== 内网环境时区测试 ===

1. 当前环境测试:
   系统时区: UTC+8 (北京时间)
   时区偏移: -480 分钟

2. 北京时间计算测试:
   系统本地时间: 2026/1/27 21:00:00
   计算的北京时间: 2026-01-27 21:00:00
   北京时间小时: 21时

4. UTC转北京时间测试:
   UTC: 2026-01-27T12:50:00.000Z
   北京: 2026-01-27 20:50:00 ✅

5. 时间一致性验证:
   时间递增: ✅ 正常
```

### 3. 启动服务

```bash
# 后端服务
cd Backed
npm install
npm start

# 前端服务
cd Unified
npm install
npm run dev
```

### 4. 验证服务

访问健康检查接口：

```bash
curl http://localhost:8000/health
```

**预期返回**：
```json
{
  "code": 0,
  "message": "ok",
  "timestamp": "2026-01-27 21:00:00",
  "timezone": "Asia/Shanghai (UTC+8)",
  "env": "development"
}
```

检查 `timestamp` 字段是否与当前北京时间一致。

## 🔧 常见问题

### Q1: 时间显示不正确怎么办？

**A**: 检查系统时间设置

```bash
# 查看系统时间
date  # Linux
Get-Date  # Windows

# 如果时间不对，手动设置
sudo date -s "2026-01-27 21:00:00"  # Linux
Set-Date -Date "2026-01-27 21:00:00"  # Windows
```

### Q2: 系统时区不是UTC+8会有问题吗？

**A**: 不会！代码会自动转换。

无论系统时区设置为什么，只要系统时间准确，代码都能正确计算北京时间。

### Q3: 定时任务会在正确的时间触发吗？

**A**: 会的！

定时任务使用 `getCurrentHour()` 函数获取北京时间的小时数，例如：
- 早餐提醒：北京时间 7:00-9:00
- 午餐提醒：北京时间 11:00-13:00
- 晚餐提醒：北京时间 17:00-19:00

### Q4: 需要定期校时吗？

**A**: 建议定期检查

虽然不需要联网校时，但建议定期检查系统时间是否准确：
- 每周检查一次
- 如果发现时间偏差，手动调整

### Q5: 数据库中的时间是什么格式？

**A**: 北京时间字符串

所有数据库中的时间字段都存储为北京时间字符串，格式为：
```
YYYY-MM-DD HH:mm:ss
例如: 2026-01-27 21:00:00
```

## 🔒 安全建议

### 1. 系统时间保护

- 限制修改系统时间的权限
- 定期检查系统时间是否被篡改
- 记录系统时间修改日志

### 2. 时间同步（可选）

如果内网有时间服务器，可以配置定期同步：

```bash
# Linux - 配置内网NTP服务器
sudo ntpdate 192.168.1.100

# Windows - 配置时间服务器
w32tm /config /manualpeerlist:192.168.1.100 /syncfromflags:manual /update
```

### 3. 监控告警

建议监控以下指标：
- 系统时间偏差（与标准时间对比）
- 定时任务执行时间
- 时间相关的异常日志
