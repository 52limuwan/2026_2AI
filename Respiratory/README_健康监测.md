# 健康监测系统使用说明

## 系统架构

```
毫米波雷达 (串口) → Python服务器 (WebSocket) → 前端页面 (实时波形)
```

## 快速开始

### 1. 启动后端服务

双击运行：`启动健康监测服务.bat`

或手动运行：
```bash
python health_monitor_server.py
```

服务将自动：
- 检测并连接毫米波雷达串口
- 启动WebSocket服务器 (ws://localhost:8765)
- 实时读取并处理生命体征数据

### 2. 访问前端页面

1. 启动前端项目（Unified目录）
2. 登录监护人账号
3. 在首页点击"健康监测"快捷入口
4. 查看实时生命体征和波形图

## 功能特性

### 实时监测
- ✓ 心率 (BPM)
- ✓ 呼吸率 (BPM)
- ✓ 检测距离 (m)
- ✓ B波（呼吸波形）
- ✓ H波（心跳波形）

### 医院风格波形图
- 实时绘制生命体征波形
- 网格背景，类似心电监护仪
- 发光效果，增强可视化
- 可独立开关B波/H波显示

### 智能警报
- 呼吸异常检测（>100 BPM）
- 呼吸偏快警告（35-100 BPM）
- 呼吸偏慢警告（<10 BPM）
- 警报历史记录

### 数据处理
- 滑动窗口降噪（5点平均）
- 异常值过滤
- 多维度生命体征判定
- 5帧/秒稳定输出

## 测试功能

在 `hx_test.py` 中可以使用快捷键测试：
- `Ctrl+O`: 模拟呼吸异常（100-150 BPM），3秒后自动恢复
- `Ctrl+P`: 模拟无生命体征，3秒后自动恢复
- `ESC`: 立即恢复正常模式

## 技术栈

### 后端
- Python 3.x
- pyserial (串口通信)
- websockets (WebSocket服务器)
- asyncio (异步处理)

### 前端
- Vue 3
- Canvas API (波形绘制)
- WebSocket (实时通信)

## 注意事项

1. **串口连接**：确保毫米波雷达已正确连接到电脑
2. **端口占用**：WebSocket默认使用8765端口，确保未被占用
3. **网络配置**：前端需要能访问 `ws://localhost:8765`
4. **数据延迟**：正常延迟约200ms（5帧/秒）

## 故障排除

### 问题：无法连接串口
- 检查雷达是否正确连接
- 确认串口驱动已安装
- 查看设备管理器中的COM端口

### 问题：WebSocket连接失败
- 确认Python服务器已启动
- 检查8765端口是否被占用
- 查看浏览器控制台错误信息

### 问题：波形不显示
- 确认Canvas元素已正确渲染
- 检查浏览器是否支持Canvas API
- 查看是否有JavaScript错误

## 开发说明

### 添加新的生命体征指标

1. 在 `health_monitor_server.py` 中添加数据处理
2. 在 `HealthMonitor.vue` 中添加显示组件
3. 更新WebSocket数据格式

### 自定义波形样式

修改 `HealthMonitor.vue` 中的 `drawWaveform` 函数：
- 网格线颜色：`ctx.strokeStyle`
- 波形颜色：传入的 `color` 参数
- 发光效果：`ctx.shadowBlur` 和 `ctx.shadowColor`
