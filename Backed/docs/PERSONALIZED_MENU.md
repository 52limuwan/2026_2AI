# 个性化菜单功能说明

## 功能概述

个性化菜单功能根据用户的健康状况（慢性病信息）自动匹配数据库中适合的菜品，并在首页展示推荐的4道菜品。

## 功能特点

1. **智能匹配**：根据用户的慢性病信息，自动匹配带有相关健康标签的菜品
2. **评分排序**：对菜品进行匹配度评分，优先推荐最适合的菜品
3. **视觉突出**：在首页顶部以卡片形式展示，配有健康标签提示
4. **快速下单**：点击菜品卡片可直接跳转到菜单页面

## 疾病与菜品标签匹配规则

系统内置了以下疾病与菜品标签的匹配规则：

| 疾病 | 推荐标签 |
|------|---------|
| 高血压 | 低盐、低钠、降压、芹菜、木耳、菌菇 |
| 糖尿病 | 低糖、无糖、粗粮、全谷物、控糖、苦瓜 |
| 高血脂 | 低脂、清淡、鱼类、豆制品、燕麦 |
| 冠心病 | 低脂、低盐、鱼类、坚果、橄榄油 |
| 痛风 | 低嘌呤、碱性、蔬菜、水果 |
| 肾病 | 低盐、低蛋白、优质蛋白 |
| 骨质疏松 | 高钙、补钙、奶制品、豆制品、虾皮 |
| 贫血 | 补铁、补血、红肉、动物肝脏、红枣 |
| 便秘 | 高纤维、粗粮、蔬菜、水果 |

## 匹配算法

1. **标签完全匹配**：菜品标签与推荐标签完全匹配，得分 +10
2. **关键词匹配**：菜品名称或描述包含推荐关键词，得分 +5
3. **排序规则**：按匹配得分从高到低排序
4. **数量限制**：最多返回4道菜品

## API 接口

### 获取个性化推荐

```
GET /client/recommendations/personalized
```

**请求参数：**
- `store_id` (可选): 店面ID，用于筛选特定店面的菜品

**返回数据：**

## 前端展示

### 首页展示位置

个性化菜单卡片显示在首页顶部，位于健康评分卡片之前。

### 展示内容

1. **标题**："为您推荐"
2. **健康标签**：显示用户的健康状况（如"高血压、糖尿病"）
3. **描述**："根据您的健康状况精心挑选"
4. **菜品网格**：2列网格布局，每个菜品卡片包含：
   - 菜品图片（或占位符）
   - 推荐理由标签（右上角）
   - 菜品名称
   - 健康标签（最多2个）
   - 价格和商户名称

### 交互功能

- 点击菜品卡片：跳转到菜单页面并高亮该菜品
- 点击"查看更多菜品"按钮：跳转到完整菜单页面

## 使用前提

用户需要在个人资料中设置慢性病信息（`chronic_conditions` 字段），系统才会显示个性化推荐。

## 测试方法

运行测试脚本：

```bash
node test-personalized-menu.js
```

测试脚本会：
1. 登录测试用户
2. 获取个性化推荐
3. 显示推荐结果和匹配详情

## 数据库字段

### client_profiles 表

- `chronic_conditions` (TEXT): 慢性病信息，JSON数组格式，如 `["高血压", "糖尿病"]`

### dishes 表

- `tags` (TEXT): 菜品标签，逗号分隔，如 `"低盐,低脂,鱼类"`
- `description` (TEXT): 菜品描述，用于关键词匹配

## 扩展建议

1. **动态规则配置**：将疾病-标签匹配规则存储到数据库，支持后台配置
2. **用户反馈**：记录用户对推荐菜品的点击和订购行为，优化推荐算法
3. **营养分析**：结合用户的营养摄入情况，推荐能补充缺失营养的菜品
4. **季节性推荐**：根据季节和节气调整推荐策略
5. **个性化权重**：根据用户历史订单偏好调整匹配权重
