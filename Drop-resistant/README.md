# 防跌倒检测系统 - 远端联动报警

## 系统架构

```
┌─────────────────────────────┐
│   检测服务端（主机）         │
│   - 连接雷达检测跌倒         │
│   - 本地报警（音效+弹窗）    │
│   - 作为服务器广播报警       │
└──────────────┬──────────────┘
               │
               │ 网络广播
               │
    ┌──────────┼──────────┐
    ↓          ↓          ↓
┌────────┐ ┌────────┐ ┌────────┐
│远端设备1│ │远端设备2│ │远端设备3│
│(护士站)│ │(监控室)│ │(家属)  │
└────────┘ └────────┘ └────────┘
```

## 文件说明

### 核心文件

1. **fall_detection_server.py** - 检测服务端
   - 连接雷达检测跌倒
   - 本地报警（音效+弹窗+可视化）
   - 作为服务器广播给远端设备
   - 运行位置：连接雷达的主机

2. **alarm_client.py** - 远端报警器
   - 接收报警并播放音效
   - 显示报警弹窗
   - 运行位置：护士站、监控室、家属设备等

3. **aviation-alarm.mp3** - 报警音效文件

### 启动脚本

- **启动检测服务端.bat** - 启动主机（雷达+服务器）
- **启动远端报警器.bat** - 启动远端设备

## 快速开始

### 步骤1：启动检测服务端（主机）

在连接雷达的主机上：

```bash
# 方式1：双击启动脚本
启动检测服务端.bat

# 方式2：命令行启动
python fall_detection_server.py
```

启动后会显示：
- 串口连接状态
- 服务器监听地址（如：0.0.0.0:9999）
- 雷达可视化界面

**获取服务器IP地址**：
```bash
# Windows
ipconfig

# 找到 IPv4 地址，例如：192.168.1.100
```

### 步骤2：启动远端报警器

在需要接收报警的设备上：

```bash
# 方式1：双击启动脚本（会提示输入服务器地址）
启动远端报警器.bat

# 方式2：命令行启动
python alarm_client.py 192.168.1.100              # 连接到指定服务器
python alarm_client.py 192.168.1.100 护士站_1号   # 指定设备名称
```

## 配置说明

### 修改服务器地址

编辑 `alarm_client.py`，修改第17行：

```python
SERVER_HOST = '192.168.1.100'  # 改为实际服务器IP
```

### 修改设备名称

编辑 `alarm_client.py`，修改第19行：

```python
DEVICE_NAME = '护士站_1号'  # 自定义设备名称
```

### 修改服务器端口

如果需要修改端口（默认9999）：

1. 编辑 `fall_detection_server.py` 第24行：
```python
SERVER_PORT = 9999  # 改为其他端口
```

2. 编辑 `alarm_client.py` 第18行：
```python
SERVER_PORT = 9999  # 改为相同端口
```

服务端广播
    ↓
护士站、值班室同时收到报警
```

### 场景2：居家养老

    ↓ 检测跌倒

    ↓
子女手机、保姆设备同时收到报警
```

### 场景3：医院病房

```
病房（雷达+主机）
    ↓ 检测跌倒
医院网络
    ↓
护士站、主治医生、监控中心同时收到报警
```

## 报警流程

1. **雷达检测** → 检测到跌倒事件
2. **本地报警** → 主机播放音效、显示弹窗
3. **网络广播** → 发送报警给所有远端设备
4. **远端响应** → 远端设备播放音效、显示弹窗

## 报警信息

报警消息包含：
- 时间：跌倒发生的时间
- 位置：X、Y坐标（米）

示例：
```
时间：2026-02-07 15:30:45
位置：X=-0.206m, Y=0.146m
```

## 网络配置

### 局域网部署（推荐）

适用于同一网络内的设备（医院、养老院）

1. 服务端自动监听所有网络接口（0.0.0.0:9999）
2. 远端设备连接到服务端的局域网IP
3. 确保防火墙开放端口9999

**Windows防火墙设置**：
```bash
netsh advfirewall firewall add rule name="FallDetection" dir=in action=allow protocol=TCP localport=9999
```

### 互联网部署

适用于远程监控（家属远程接收）

需要：
- 服务器有公网IP或使用内网穿透
- 开放服务器端口9999
- 远端设备连接到公网地址

## 故障排查

### 问题1：远端设备无法连接

**检查**：
1. 服务端是否已启动
2. 服务器IP地址是否正确
3. 防火墙是否开放端口
4. 网络是否连通（ping测试）

### 问题2：无法播放音效

**检查**：
1. 是否安装pygame：`pip install pygame`
2. 音效文件是否存在
3. 音频设备是否正常

### 问题3：串口无法打开

**检查**：
1. 雷达是否正确连接
2. 串口是否被其他程序占用
3. 是否有串口访问权限

## 远端报警器命令

启动后可输入以下命令：

- `status` - 查看连接状态
- `test` - 测试报警音效
- `stop` - 停止报警音效
- `quit` - 退出程序
